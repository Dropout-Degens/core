# Supabase Query Templates Library\n\n**Status:** Open  \n**Date Created:** 2026-03-05  \n**Priority:** High  \n**Owner:** Dax (requirements), YodAI (implementation)\n\n## Problem\nCurrently, every Supabase query is executed live without reference or reusability. This creates:\n- Repetitive work (same query asked multiple ways)\n- No institutional knowledge of what queries are useful\n- Inconsistent data retrieval patterns\n- No way to optimize or audit frequently-used queries\n- Lost context on what was queried and when\n\n## Solution\nBuild a **query templates library** stored in `context/operations/data/` that:\n1. Catalogs frequently-used Supabase queries\n2. Organizes by purpose (user analytics, bet trends, revenue, etc.)\n3. Allows YodAI to reference and reuse them consistently\n4. Serves as documentation of what data is available and how to access it\n\n## What Needs to Happen\n\n### 1. Audit Recent Queries\n**Before building templates, identify what's actually been queried:**\n- Check YodAI logs/session history for all Supabase queries executed in last 30 days\n- Document:\n  - Query text (exact SQL)\n  - Date/time executed\n  - Purpose (what question was being answered)\n  - Schema(s) used (private, manual, dax-personal)\n  - How often repeated (if at all)\n  - Results / findings\n\n**Output:** `context/operations/data/query-audit-log.md`\n- Chronological list of all recent queries\n- Grouped by schema\n- Flagged if executed multiple times with variations\n\n### 2. Create Template Format\n\n**File structure:**\n```\ncontext/operations/data/\n  query-templates/\n    user-analytics.md\n    bet-trends.md\n    revenue-metrics.md\n    event-data.md\n```\n\n**Template format example:**\n```markdown\n# User Signups Over Time\n\n**Purpose:** Track daily/weekly new user acquisition rate\n**Schema:** private\n**Frequency:** Weekly check-in\n\n## Query\n\\`\\`\\`sql\nSELECT \n  DATE(\"createdAt\") as signup_date,\n  COUNT(*) as new_users\nFROM private.\"users\"\nGROUP BY DATE(\"createdAt\")\nORDER BY signup_date DESC\nLIMIT 30\n\\`\\`\\`\n\n## Variables\n- `LIMIT 30` → adjust for different date range\n- Optional: add `WHERE \"status\" = 'active'` to filter\n\n## Last Used\n2026-03-03 (found X new signups)\n\n## Notes\n- Uses private.users table\n- Timezone-aware (depends on DB config)\n- Good for tracking acquisition trends\n```\n\n### 3. Organize by Category\n\n**Suggested categories (adjust based on audit):**\n- **User Analytics** — Signups, active users, retention, churn\n- **Bet Data** — Placed bets, outcomes, hit rates, win/loss streaks\n- **Revenue Metrics** — Transactions, income streams, costs\n- **Event Data** — Alerts, tickets, giveaways, platform activity\n- **Membership & Subscriptions** — Active members, tier breakdown, churn\n- **Custom Queries** — One-off analyses that became useful\n\n### 4. Update Supabase README\n\n**Current README should link to:**\n- Query templates library location\n- How to use a template (copy, modify vars, run)\n- When to create new template vs. custom query\n- Rules for each schema (who can read, who can write)\n\n**Add section:**\n```markdown\n## Query Templates\n\nFrequently-used queries are stored in `context/operations/data/query-templates/`.\n\n**To use a template:**\n1. Find relevant category in query-templates/\n2. Copy the SQL\n3. Adjust variables (dates, limits, filters) as needed\n4. Execute via `query_private`, `query_personal`, or `query_manual`\n\n**To add a new template:**\n- If you find yourself writing the same query multiple times, document it\n- Include purpose, variables, last used date, and notes\n- Update the index below\n```\n\n### 5. Give YodAI Template Access\n\n**In my instructions/capabilities:**\n- Before executing a Supabase query, check if there's a template in `context/operations/data/query-templates/`\n- If yes: reference the template, show which one I'm using\n- If no: execute as custom query, but flag if it seems like something worth templating\n- At end of day, suggest new templates for frequently-run queries\n\n**Example interaction:**\n```\nYou: \"How many new users signed up this week?\"\n\nMe: \"Found template: User Signups Over Time. Running...\n[executes template with LIMIT adjusted to 7 days]\nResult: X new users\"\n```\n\n## Dependencies\n- Audit of recent queries must happen first\n- Supabase README must exist and be accessible\n- YodAI needs read access to `context/operations/data/query-templates/`\n\n## Implementation Checklist\n\n- [ ] Audit YodAI logs for all Supabase queries (last 30 days)\n- [ ] Create `query-audit-log.md` documenting findings\n- [ ] Identify top 10-15 most useful or repeated queries\n- [ ] Create `context/operations/data/query-templates/` directory\n- [ ] Write templates for high-value queries\n- [ ] Organize templates by category\n- [ ] Create index/README in query-templates/ folder\n- [ ] Update main Supabase README to reference templates\n- [ ] Test template retrieval and execution\n- [ ] Update YodAI instructions to check templates before custom queries\n- [ ] Add \"suggest new template\" to daily recap review\n\n## Success Signal\n\n- Can answer common data questions by referencing a template (faster response)\n- New data questions identify missing templates (captured for next iteration)\n- Query audit log shows patterns (e.g., \"this query runs weekly, worth templating\")\n- Supabase README is up-to-date and points to templates\n- YodAI's queries are consistent and reusable across sessions\n\n## Next Steps\n\n1. **Immediate:** Audit logs and capture recent queries\n2. **Short-term:** Build template library from top queries\n3. **Ongoing:** Update templates as new patterns emerge, add new templates quarterly\n
